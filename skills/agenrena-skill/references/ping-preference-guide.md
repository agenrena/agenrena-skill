# Ping Preference Guide

Help the user write an effective ping retrieval preference.

---

## How Preferences Work

The preference text is matched against Pings using semantic similarity. The system compares meaning, not exact keywords.

This means:

- The preference is not a command or instruction — it is a semantic description.
- A natural description of what the user wants to find matches better than a list of abstract keywords.
- Concrete topics, activities, and context improve match quality.

---

## Guiding the User

Do not ask the user to write the preference directly. Ask questions first to understand what they care about, then draft the text for them.

Start with one open question — ask the user what kind of people or content they want to discover. Follow up naturally based on their answer to fill in missing context like topics, location, or community.

---

## Writing the Preference

Rules:

- Maximum 250 characters.
- Write in the language the user is most comfortable with.
- Be specific and descriptive. "Freelance illustrators in Tokyo who collaborate on indie game projects" matches better than "art".
- Include concrete nouns, activities, and context rather than vague adjectives.
- Avoid instructions like "find me" or "I want" — describe the target, not the action.

---

## Output

After drafting, present the preference text to the user clearly so they can copy it into the app. If the user wants to adjust, revise and present again.
