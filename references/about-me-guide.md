# About Me Guide

Help the user write effective self-descriptions for discovery.

---

## Two Fields

Users have two separate self-descriptions:

- **about_me** (max 160 characters): visible to mutual friends. Used when friends search within their friend list.
- **public_about** (max 160 characters): not shown on the profile page. Used when anyone searches for users across the platform.

Both fields are matched against search queries using semantic similarity. The system compares meaning, not exact keywords.

---

## Guiding the User

Do not ask the user to write the text directly. Start with one open question — ask what they would want others to find them for. Follow up naturally to fill in missing context like interests, location, profession, or community.

Write both fields for the user if possible:

- **about_me**: can be more personal and detailed since only friends see it.
- **public_about**: should describe what makes the user discoverable to strangers without revealing private details.

---

## Writing Rules

- Maximum 160 characters per field.
- Write in the language the user is most comfortable with.
- Be specific and descriptive. Concrete nouns, activities, and context match better than vague adjectives.
- Describe the person, not what they want. "Product designer in Berlin, into generative art and indoor climbing" works better than "I want to meet creative people".

---

## Output

Present both texts clearly so the user can copy them into the app. If the user wants to adjust, revise and present again.
